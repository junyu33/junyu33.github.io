从一道 leetcode 题到离散对数求解算法

发表于 2025-03-18 分类于 笔记 阅读次数： 本文字数： 9.3k 阅读时长 ≈ 8 分钟

我们先来看题面，点进去之前可以先思考一下：

Given an array of integers nums containing n+1n + 1n+1 integers where each integer is in the range [1,n][1, n][1,n] inclusive, n≤105n \le 10^5n≤105.

There is only one repeated number in nums, return this repeated number.

You must solve the problem without modifying the array and using only constant extra space.

算法题解答

如果不看第三行，那么任意O(nlog⁡n)O(n \log n)O(nlogn)的排序都可以做这道题。即使不限制 O(1)O(1)O(1) 的空间的话，因为元素在 [1,n][1, n][1,n] 之间，桶排序也是可以 AC 的。但如果既不能修改数组 nums，也不能新开数组，那就只能考虑双指针解法了。

一个比较容易想到的做法是 binary search，既然值域在 [1,n][1, n][1,n]，我们可以令 mid=⌊1+n2⌋mid = \lfloor\frac{1+n}{2}\rfloormid=⌊21+n​⌋，然后取 num 中 的元素的个数（记为xxx），如果 x≤midx \le midx≤mid。我们可以证明 [1,mid][1, mid][1,mid] 之间没有重复元素：

这里使用反证法，记[condition][condition][condition]为当“condition”为真时值为111，否则为000。

假设[1,mid][1, mid][1,mid] 之间存在重复元素且 ∑i=1n[numi≤mid]=x\sum_{i=1}^n [num_i \le mid] = x∑i=1n​[numi​≤mid]=x。因为重复元素只有一个，因此[mid+1,n][mid+1, n][mid+1,n]这些元素只会存在000个或者111个。因此 ∑i=1n[numi>mid]≤n−mid\sum_{i=1}^n [num_i > mid] \le n - mid∑i=1n​[numi​>mid]≤n−mid，两式相加可得 n+1≤x+n−mid≤nn+1 \le x + n - mid \le nn+1≤x+n−mid≤n，矛盾！

因此假设不成立，原结论正确，[1,mid][1, mid][1,mid] 之间没有重复元素。

有了上述的结论，我们可以断定当x≤midx \le midx≤mid时，应当继续在 [mid+1,n][mid+1, n][mid+1,n] 中寻找。而当x>midx > midx>mid时，根据抽屉原理显然 [1,mid][1, mid][1,mid] 之间有重复元素，应当在 [1,mid][1, mid][1,mid] 中继续搜索。

以上过程递归进行，总时间复杂度为 O(nlog⁡n)O(n \log n)O(nlogn)。

Can you solve the problem in linear runtime complexity?

然后算法的下一步为了找到环的“起始点”（也就是点 5。如果他们一开始就在环里那自然就是起点了）。将 tortoise 放回了起点 4，hare 保持原先位置 3 不动，随后它们每次走一步：

可以发现它们在环的“起始点” 5 处相遇了。可以证明，无论图的形状如何，它们始终都会在环的“起始点”处相遇：

设环的周长为CCC，尾巴（也就是图中 5->6->4 的长度为 ddd）当他们第一次相遇时，hare 走了 2t2t2t步，tortoise 走了 ttt 步，显然我们有 C∣(2t−t)C | (2t - t)C∣(2t−t)。而当 tortoise 被移回起点时，由于此时它们速度一样，hare 和 tortoise 的距离就恒为 2t2t2t。当 tortoise 走了 ddd 步到了“起始点”时，hare 走了 2t+d2t+d2t+d 步，由于 C∣2tC | 2tC∣2t，因此 hare 和 tortoise 必相遇，并且都在“起始点”。

算法的最后一步就是 hare 不动，tortoise 绕一圈测出周长CCC，这个算法就结束了。

回到本题，由于重复的数字只有一个，所以肯定是“起始点”本身，因此我们只需要做算法的前两步即可 AC。时间复杂度 O(n)O(n)O(n)。

Brent’s algorithm

Brent 在 Floyd 的探圈算法基础上进行了改进，使用了 2 的幂次步长power来检测循环。并引入了lambda（每次迭代自增，可以理解为周长CCC）和 mu（自增，最后得出尾巴 ddd 的值）。该算法的优势有两点：

  • 直接在第一步找到了环的周长CCC。
  • 每一次迭代只需要计算一次fff，而不是 floyd 的三次。

  • Frame 6: Tortoise at 4, Hare at 4, power = 4, lambda = 3 (cycle length detected).
  • Frame 7: Tortoise at 1, Hare at 1 (reset for finding position of length lambda).
  • Frame 8: Tortoise at 1, Hare at 2.

  • Frame 11: Tortoise at 2, Hare at 5, mu = 1.

pollard_rho

由于这个建图很容易让人想起ρ\rhoρ这个希腊字母，刚好 Pollard 本人也是这么想的。于是他找到了一个代数结构：有限环RnR_nRn​，然后加了一个运算 f(x)=(x2+t)mod  nf(x) = (x^2+t) \mod nf(x)=(x2+t)modn 用来生成一个伪随机序列 A=[x,f(x),f2(x),⋯ ]A = [x, f(x), f^2(x), \cdots]A=[x,f(x),f2(x),⋯]。如果 n=pqn=pqn=pq，那么我们假设这个随机序列在[Aimod  p][A_i \mod p][Ai​modp]中均匀分布，根据生日悖论，我们期望在 O(p)O(\sqrt{p})O(p​)的时间内找到一对相同的值。

但我们事先不知道ppp的值是多少，因此需要通过 gcd⁡(∣Ai−Aj∣,N)\gcd(|A_i-A_j|, N)gcd(∣Ai​−Aj​∣,N) 是否不为1来判断。因为如果 Ai≡Ajmod  pA_i \equiv A_j \mod pAi​≡Aj​modp，显然 ∣Ai−Aj∣|A_i-A_j|∣Ai​−Aj​∣ 就是 ppp 的倍数，从而 gcd⁡(∣Ai−Aj∣,N)\gcd(|A_i-A_j|, N)gcd(∣Ai​−Aj​∣,N) 就不为1。

那么我们如何来遍历这里的 i,ji,ji,j（hare and tortoise）呢，虽然序列AAA是 mod  n\mod nmodn 的，但考虑Aimod  pA_i \mod pAi​modp 与 Ajmod  pA_j \mod pAj​modp， 因为FpF_pFp​是一个域，对一个元素反复做运算fff，一定会形成一个周期（或者叫“轨道”）。这样的话，Floyd’s cycle-finding 就派上用场了，在经过有限次的迭代后，可能在 FpF_pFp​ 中 i,ji,ji,j 先相遇返回结果ppp。同理，或者在 FqF_qFq​ 中先相遇返回结果qqq。或者也有可能会在 RnR_nRn​ 中相遇返回结果nnn，然而这个结果是平凡的，需要重新执行该算法。

这个算法便称为 pollard_rho 素性检测，因为 ppp 在最坏的情况是 O(n)O(\sqrt{n})O(n​) 的，因此这个时候 pollard_rho 的平均时间复杂度为 O(n1/4)O(n^{1/4})O(n1/4)。

pollard_rho for DLP

就像将质因数分解问题联想到 DLP（离散对数问题）那样自然一样，除了在有限环RnR_nRn​中做 pollard_rho 以分解质因数nnn之外，Pollard 本人还将该方法论尝试在 DLP 中得到应用。

DLP 与 pohlig-hellman 算法结合

pohlig-hellman 算法也可以用于高效求解 DLP 问题，主要思路将 n−1n-1n−1 进行质因数分解，然后使用某种算法（链接里使用了 BSGS）来进行GF(p)\mathrm{GF}(p)GF(p)中离散对数的求解，最后用 CRT（中国剩余定理）合并各个质数ppp的结果。具体的介绍阮行止写得很好，可以去看一看：

https://www.ruanx.net/pohlig-hellman/

若知道 DLP 中 n−1n-1n−1 的质因数分解方法，则可以使用该算法与先前的 pollard_rho 算法结合，将 DLP 求解时间复杂度从 pollard_rho 的 DLP 版本的时间复杂度从O(n)O(\sqrt{n})O(n​)降低至O(p)O(\sqrt{p})O(p​)，其中 ppp 是 n−1n-1n−1 的最大质因数。

因此综上所述 DLP 这个问题取决于 n−1n-1n−1 本身是否光滑，对于通用 nnn（例如大素数），时间复杂度是Pollard’s Rho 或 BSGS 的 O(n)O(\sqrt{n})O(n​)，如果 nnn 已分解为∏piei\prod p_i^{e_i}∏piei​​，则时间复杂度为 O(∑eipi)O(\sum e_i \sqrt{p_i})O(∑ei​pi​​)。
